# Kicktipp WC 2026 — Sigurando Bitak Dashboard

Static GitHub Pages dashboard for the Kicktipp.de World Cup 2026 prediction league.

## Quick Start

```bash
# Build data + deploy to gh-pages in one command
./scripts/update.sh

# Build only (no deploy)
python3 scripts/build_data.py

# Build + commit, but don't push
./scripts/update.sh --no-push
```

## How to Refresh After New Matches

When a match result changes (e.g., **Spain 2:0 France**):

1. **Download fresh zip exports** from Kicktipp.de
   - Log in, go to your league, export all predictions/overview files
   - Replace the `*.csv.zip` files in this repo root

2. **Run the update script**
   ```bash
   ./scripts/update.sh
   ```
   This will:
   - Re-download actual match results from the [GitHub dataset](https://github.com/mominullptr/FIFA-World-Cup-2026-Dataset)
   - Rebuild `site/data/data.js` with new points and stats
   - Switch to `gh-pages` branch, copy files, commit & push

3. **Done** — GitHub Pages updates automatically within ~1 minute.

### If the GitHub dataset hasn't been updated yet

The script caches results in `data/results_cache.json`. If the download fails, it falls back to the cache automatically. To force use of the cache:

```bash
python3 scripts/scrape_results.py --cached
```

## Data Sources

| Source | What it holds | Updated by |
|---|---|---|
| `*.csv.zip` | Kicktipp predictions + overview points | You (manual download) |
| `matches_detailed.csv` (GitHub) | Actual match results, top scorers | [mominullptr](https://github.com/mominullptr/FIFA-World-Cup-2026-Dataset) |
| `data/results_cache.json` | Cached match results (fallback) | Auto-saved by `scrape_results.py` |

## Project Structure

```
kicktipp/
├── scripts/
│   ├── scrape_results.py    # Fetch + cache match results
│   ├── build_data.py        # Extract zips, compute stats → data.js
│   └── update.sh            # One-command build + deploy
├── data/
│   └── results_cache.json   # Cached match results
├── site/                    # Static site source (master branch)
│   ├── index.html
│   ├── css/style.css
│   ├── js/app.js
│   └── data/data.js         # Generated by build_data.py
└── *.csv.zip                # Kicktipp exports (tracked in git)
```

The `gh-pages` branch contains only the flat site files (`index.html`, `css/`, `js/`, `data/`) for GitHub Pages.

## Scoring Rules

| Scenario | Points |
|---|---|
| Exact score (incl. exact draw) | 4 |
| Correct winner + correct goal difference | 3 |
| Draw predicted (no exact) | 3 |
| Correct winner only | 2 |
| Bonus: WC winner correct | 10 |
| Bonus: Top scorer correct | 10 |

## Tech Stack

- **ECharts** — interactive charts (dark theme, animations)
- **Vanilla JS** — no framework, no build step
- **Python 3** — data extraction and build scripts
- **GitHub Pages** — hosting from `gh-pages` branch
